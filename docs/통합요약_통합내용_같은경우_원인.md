# 통합 요약과 통합한 내용이 같은 경우 — 원인 분석

## 흐름 요약

| 단계 | 통합한 내용 (merged_content) | 통합 요약 (merged_summary) |
|------|-----------------------------|----------------------------|
| 1 | 그룹 기사 1건 → `body_snippet` 또는 `summary` 또는 제목 | (통합한 내용 생성 후) |
| 2 | 그룹 기사 2건 이상 → Gemini `merge_content()` (합집합 본문, 15문장·800자 이상) | — |
| 3 | 없으면 fallback: 첫 기사 `body_snippet` / `summary` / 제목 | — |
| 4 | `_ensure_paragraph_breaks()` 적용 | — |
| 5 | — | `summarize(client, merged_content, max_chars=500)` 호출 |
| 6 | — | 실패 시 `_first_sentence(merged_content)` 등 fallback |

- **통합 요약**은 항상 **통합한 내용을 입력으로** `summarize()`를 호출해 만듭니다.
- `summarize()` 프롬프트: *"다음 뉴스 본문을 3문장 이내로 요약해 주세요. 핵심 수치·날짜는 유지하세요."*
- 입력은 최대 8000자까지 잘라서 전달됩니다.

---

## 왜 같은 값이 나오는가

### 1. 단일 기사 그룹 + 이미 요약문이 통합한 내용인 경우 (가장 흔한 원인)

- **통합한 내용**  
  - 기사가 1건일 때: `merged_content = body_snippet or summary or title`  
  - Google News 등에서 `body_snippet`이 비거나 짧으면 **이미 기사 요약(summary)**이 그대로 들어갑니다.  
  - 예: 115자, 135자 같은 **이미 3문장 이내 요약문**.

- **통합 요약**  
  - `summarize(client, merged_content)`  
  - 입력이 **원래부터 짧은 요약문**이면, 모델이 “이미 요약된 글”로 보고 **거의 그대로 반환**하는 경우가 많습니다.  
  - 그래서 **통합 요약 ≈ 통합한 내용**이 됩니다.

정리하면, **통합한 내용에 “본문”이 아니라 “요약(summary)”이 들어간 경우**, 그걸 다시 요약하라고 해도 결과가 비슷하거나 같아질 수 있습니다.

### 2. 통합한 내용 자체가 이미 짧은 경우 (2~3문장)

- `merged_content`가 애초에 2~3문장(예: 200자 미만)이면,  
  “3문장 이내로 요약”해도 **문장 수가 그대로**이거나 약간만 바뀌어서,  
  문자열이 같거나 매우 비슷하게 나올 수 있습니다.

### 3. 여러 기사 그룹인데 merge_content 실패 후 fallback인 경우

- 기사가 2건 이상이어도 `merge_content()`가 실패하면  
  `merged_content = 첫 기사 body_snippet or summary or title` 로 채워집니다.  
  - 이 fallback이 **짧은 요약문**이면, 위 1번과 같은 이유로  
  - `summarize(merged_content)` 결과가 **통합한 내용과 같거나 비슷**해질 수 있습니다.

### 4. 여러 기사 그룹 + merge_content 정상인 경우

- `merge_content()`가 15문장·800자 이상으로 잘 나오면,  
  `summarize()`는 이를 3문장 이내로 줄이므로 **보통 통합 요약 ≠ 통합한 내용**입니다.  
  즉, “같은 경우”는 주로 **단일 기사**이거나 **통합한 내용이 짧은 요약일 때** 발생합니다.

---

## 결론

| 조건 | 통합 요약 = 통합한 내용이 나오기 쉬운 이유 |
|------|--------------------------------------------|
| 그룹 1건 + body 없음 | 통합한 내용 = 기사 `summary`(이미 요약) → 요약 again → 그대로 반환 |
| 통합한 내용이 2~3문장 | “3문장 이내 요약”에 맞춰 변경 여지가 적음 |
| merge_content 실패 후 fallback | 통합한 내용이 짧은 요약 → 위와 동일 |

**핵심**:  
- **통합한 내용**에 **본문(body_snippet)**이 충분히 들어가면, 통합 요약은 보통 “짧게 압축된 다른 텍스트”가 됩니다.  
- **통합한 내용**이 **이미 요약(summary)**이거나 **매우 짧은 텍스트**일 때, 통합 요약과 통합한 내용이 같거나 거의 같게 나옵니다.

---

## 개선 방향 (선택)

- **통합한 내용이 이미 짧을 때** (예: 300자 미만)  
  - `summarize()`를 건너뛰고 `merged_summary = merged_content`로 두거나,  
  - “이미 요약된 글은 요약하지 말고 그대로 쓴다”는 지시를 프롬프트에 넣는 방식으로,  
  “의도적으로 같은 경우”를 명확히 할 수 있습니다.  
- 또는 **단일 기사일 때** 통합한 내용에 `body_snippet`을 우선 채우도록 수집/저장을 보강하면,  
  통합한 내용이 길어져서 통합 요약과 달라질 가능성이 높아집니다.
