# 뉴스 에이전트 — 프론트엔드 프로토타입

서버 없이 **HTML만**으로 동작하는 화면 프로토타입입니다. **처음 접속 시 로그인(비밀번호 입력) 후에만 피드를 볼 수 있습니다.**

## 실행 방법

1. **로컬에서 파일로 열기**  
   `index.html` 또는 `login.html`을 열면 **로그인 화면**이 나옵니다. 비밀번호를 입력하고 로그인하면 피드로 이동합니다.

2. **간단 로컬 서버 (선택)**  
   ```bash
   cd prototype
   python3 -m http.server 8080
   ```
   브라우저에서 `http://localhost:8080/` 접속 → 자동으로 로그인 페이지로 이동 → 비밀번호 입력 후 로그인 → 피드 이용 가능.

## 로그인 동작

- **첫 화면**: `index.html`은 곧바로 `login.html`로 이동합니다. 항상 비밀번호 입력 화면부터 시작합니다.
- **로그인**: 비밀번호를 입력하고 "로그인"을 누르면 피드(`feed.html`)로 이동합니다. (프로토타입에서는 비밀번호 검증 없이 입력만 하면 됩니다.)
- **보호된 페이지**: 피드, 검색, 기사 상세, 그룹 상세는 **로그인하지 않으면 접근할 수 없습니다.** 미로그인 상태에서 해당 주소로 가면 로그인 페이지로 돌려보냅니다.

## 구성

| 파일 | 설명 |
|------|------|
| `index.html` | 진입 시 로그인 페이지(`login.html`)로 자동 이동 |
| `login.html` | **로그인 화면** — 비밀번호 입력 후 피드로 이동 |
| `feed.html` | 피드(목록): 그룹·기사 카드, 필터 바, 검색창 |
| `search.html` | 검색 결과 (쿼리 `?q=검색어` 반영) |
| `article.html` | 기사 상세: 제목, 요약, 원문, **관련 뉴스 히스토리**, 편집 모달 |
| `group.html` | 통합 뉴스 상세: 통합 요약, 같은 뉴스 묶음, **관련 뉴스 히스토리**, 통합 요약 편집 |
| `css/app.css` | 공통 스타일 |
| `js/mock-data.js` | 목업 데이터 (피드·기사·그룹·연관 히스토리) |
| `js/auth.js` | 로그인 체크 (sessionStorage, 보호된 페이지에서 사용) |

## 데이터

- 모든 데이터는 **`js/mock-data.js`**에 있습니다. 화면을 바꿔보고 싶으면 이 파일의 `MOCK_DATA`를 수정하면 됩니다.
- 편집·저장은 **메모리에서만** 반영됩니다. 새로고침하면 원래 mock 데이터로 돌아갑니다.

## 참고
